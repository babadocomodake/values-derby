# お題設計ガイド（価値観ダービー）

このファイルは「良いお題とは何か」を言語化した**設計の物差し**。
公式お題の刷新・追加お題の生成・JSON検証のすべてがここを基準にする。
（迷ったら必ずここに戻る。完璧を最初から狙わず、実プレイで反復して育てる）

---

## 0. 北極星（一文）

> **軽く楽しく答えられるのに、順位をつけると本人の好み・価値観・人間性が透けて、
> 思わず「なんで？」と会話したくなる絶妙な問い。**

重すぎる問い（死生観・政治）でも、薄すぎる問い（朝ごはんは何派）でもない、
その**真ん中の“絶妙ゾーン”**だけを狙う。

```
  薄い ───────────●─────────── 重い
 朝ごはん？     ココが狙い      死をどう思う？
 （価値観透けない） （楽しい×価値観） （楽しくない/気まずい）
```

---

## 1. 良いお題の5条件（採否チェックリスト）

1問つくるたびに、この5つに○がつくか確認する。3つ以上欠けたらボツ。

| # | 要素 | 判定の問い |
|---|------|-----------|
| ① 投影性 | 順位で**その人らしさ**が出るか | 「この人こういう人なんだ」と分かる？ |
| ② 葛藤 | **少し悩む**か（拮抗） | 7つが拮抗して、1位が一瞬迷う？ |
| ③ 会話喚起 | **「なんで？」と聞きたくなる**か | 順位を見たら理由を尋ねたくなる？ |
| ④ 軽さ・安全 | 誰でも30秒・誰も傷つかないか | 家族/友人で気まずくならない？ |
| ⑤ 割れる | 人によって**答えがバラける**か | 全員同じ順位になりそう＝ボツ |

※「正解がある（知識クイズ）」は①も⑤も死ぬので即ボツ。

## 2. 設問の型カタログ（投影が起きる質問フォーマット）

価値観が透ける質問には“型”がある。新しいお題はまずこの型から発想する。

| 型 | ねらい（何が透けるか） | 例 |
|----|----------------------|----|
| **願望型**（〜されたら嬉しい／欲しい） | 何を**求める**人か | 「言われたら一番嬉しい言葉は？」優しい/賢い/面白い/可愛い/かっこいい… |
| **回避型**（〜は嫌／されたら困る） | 何を**恐れ・守る**か | 「恋人にされたら一番イヤなのは？」 |
| **喪失型**（なくなったら困る） | 何を**大事**にしているか | 「世の中から消えたら困る施設は？」公園/図書館/遊園地… |
| **トレードオフ型**（全部は手に入らない） | 価値の**優先順位** | 「結婚相手に求めるなら？」収入/性格/価値観/外見… |
| **投影型**（比喩・たとえ＝心理テスト由来） | 本音を**間接的**に | 「無人島に1つ持っていくなら？」「生まれ変わるなら何の動物？」 |
| **if/仮定型**（もし〜だったら） | 欲望・行動の**クセ** | 「宝くじで1億当たったら最初に？」 |
| **理想自己型**（どう思われたい／どう在りたい） | **自己像**・コンプレックス | 「お葬式で言われたい一言は？」 |

> 出典の発想源: 「言われたら嬉しい言葉」「なくなったら困るもの」「家の近くにあったら嫌なもの」（ユーザー提示の良例）＋
> 心理テストの**投影法**（無人島・動物・色・家など、直接聞かず“たとえ”で本音を引き出す手法）。

## 3. 心理テストから抜いた「投影の仕掛け」（なぜ本音が出るか）

追加お題は心理テストを発想源にするので、その**効く理由**を要素化しておく。

- **間接化**: 「あなたの価値観は？」と直接聞かず、モノ・場面・たとえに**乗せて**聞く → 構えずに本音が出る。
- **強制順位づけ**: 「どれも良い」を許さず**順位を強制**する → 普段意識しない優先順位が露わになる。
- **拮抗する選択肢**: 明らかな勝者/敗者を作らない → **迷い＝その人の判断基準**が見える。
- **理由が割れる前提**: 同じ1位でも理由が人それぞれ → 「なんで？」で会話が爆ぜる。
- **ポジ/ネガ両面**: 「欲しい」だけでなく「嫌」も使う → 嫌いの順位は好き以上に人間性が出る。

---

## 4. 選択肢7つの作り方（重要：ここで質が決まる）

設問が良くても選択肢が雑だと死ぬ。7択は次を満たす。

- **拮抗**: 「明らかな1位」「明らかな最下位」を作らない。7つが横並びで悩ましい。
- **軸を散らす**: 似たもの同士で埋めない（例: 食べ物なら甘い/しょっぱい/ヘルシー/背徳…と方向を変える）。
- **具体・イメージ可能**: 抽象語より絵が浮かぶ言葉（「自由」より「平日の昼に何もしない時間」）。
- **社会的正解を作らない**: 「優しさが1位に決まってる」みたいな“模範解答”を避ける。
- **ちょうど7つ**: 採点ロジックが7択固定。多すぎ/少なすぎは検証で弾く。

## 5. アンチパターン（作ってはいけないお題）

- ❌ **表層トリビア**: 朝ごはんパン派/ご飯派 → 価値観が透けない。
- ❌ **重い/センシティブ**: 死・政治・宗教・収入額・容姿いじり → 楽しくない／気まずい。
- ❌ **知識クイズ**: 正解がある → 当てるゲームになり価値観が消える。
- ❌ **答えが全員同じ**: 「人を殺すのは良いこと？」的に割れない → ゲームにならない。
- ❌ **抽象すぎ**: 「幸せとは？」 → 答えにくく会話も散る。

---

## 6. ジャンル・トーンの分散軸（デッキ生成で使う）

1ゲームのデッキは**ジャンルが偏らないよう散らす**。型(types.ts)に対応。

- **Genre**: food（食/嗜好）/ love（恋愛・人間関係）/ money（お金・損得）/ life（人生・生き方）/ dilemma（究極の二択的ジレンマ）/ daily（日常の小さな選択）/ other。
- **Tone**: light（笑える・軽い）/ mid（少し考える・本音が出る）。重い(heavy)は採用しない。

## 7. お題1問のデータ形（JSON）と検証ルール

```jsonc
{
  "id": "off-001",        // 公式=off- / 追加=add- で接頭辞を分ける（要件①）
  "q": "言われたら一番うれしい言葉は？",
  "opts": ["優しいね","頭いいね","面白いね","かわいいね","かっこいいね","センスいいね","一緒にいて楽"], // ちょうど7つ
  "genre": "love",
  "tone": "light",
  "source": "official"     // official | custom（要件①②の出し分けに使う）
}
```

**検証（npm script / テストで強制）**:
- `id` が全体で重複しない。
- `opts` がちょうど7要素・空文字なし・重複なし。
- `genre` / `tone` が enum に一致。
- `source` が official | custom のいずれか。

## 7.5 実カードから抽出した型・傾向（← 追加カード作成の燃料）

公式の実物カード（docs/official-cards.md）を**ジャッジするためでなく、効いている仕掛けを盗んで
追加カード(EXTRA)に活かすため**に分析する。カードを取り込むたびにここへ知見を追記する。

### 抽出した「型」（§2カタログに追加で使う）
- **制約下コーピング型**: 「〇〇できない状況で何を持つ/する？」例: 1週間外出禁止のホテルに持っていくのは？
  → 暇・孤独の埋め方で内面が出る。応用: 「停電で1日スマホ無し、何して過ごす？」
- **ネガ代償ジレンマ型**: 「一生〜できない、どれなら許せる？」例: 1億円もらえる代償。
  → ジレンマの“具体縛り”版。応用: 「タイムマシンの代償、どれを失う？」
- **関係を機能で選ぶ型**: 「今ほしい友達／人は？」例: 今、欲しい友達は？
  → 人間関係への本音（打算）が出て会話が弾む。応用: 「チームに1人足すなら？」
- **喪失逆型（具体物版）**: 「世界からなくなっても困らないのは？」
  → 「何が要らないか」で価値観が透ける。§2「喪失型」の逆向き。
- **痛点型（ネガ感情の最上位）**: 「立ち直れないことは？／一番ショックなのは？」
  → 何に一番傷つくか＝何を大事にしているか（推し/愛/お金/承認/思い出）。共感とツッコミが両立。
- **好奇心の向き先型**: 「タイムマシンで見学したいのは？／専門家になるなら？」
  → 過去/未来/自分/知識のどこに心が向くかで知的志向が出る。軽くて会話が弾む。

### 実カードに共通する「効く傾向」（追加カードの作り方ガイド）
1. **マジメ⇔ネタの振り幅**を選択肢に持たせる（例: 奇祭専門家／激ムズの知恵の輪）→ 笑いと本音を両立。
2. **固有名詞・数字で具体化**（600ページ／激ムズ／夜12時）→ 情景が浮かび、選びやすく覚えやすい。
3. **問いを日常から少しズラす仮定**（外出禁止／1億円／世界消滅／専門家になる）→ 構えず本音が出る。
4. **ネガ方向を積極活用**（嫌／困らない／できない）→ 「好き」より人柄が濃く出る。
5. **選択肢は短語でなく“文”でもよい**（〜な人／一生〜できない）→ ただし**UIの長文対応が前提**。

> 使い方: 新しい追加カードを作るときは、まず上の「型」から1つ選び、「効く傾向」を2つ以上盛り込む。

## 8. 運用メモ

- まず**ざっくり作って実プレイ**→「会話が起きたか」で残す/捨てるを判断（北極星§0の③で評価）。
- 良かったお題は**型(§2)を記録**して横展開（同じ型で別ジャンルを量産）。
- このガイドは生き物。実プレイで気づいた要素は随時ここへ追記する。
